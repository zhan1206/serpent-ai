# serpent-ai CI 修复进度 - 2026-05-30

## 已完成修复（commit da7159b）

**根因**：CI Linux 环境下多个 `backend/` 模块存在第三方库裸导入，`pip install` 时若这些库安装失败（如缺少系统依赖），Python 会在模块导入时崩溃，导致 pytest 根本无法运行。

**修复内容**：将所有第三方库裸导入改为 try-except 可选导入（不存在时设为 None）：

|------|------|
| `backend/models/openai_adapter.py` | `import openai` → try-except |
| `backend/models/anthropic_adapter.py` | `import anthropic` → try-except |

| `backend/models/doubao_adapter.py` | `import openai` → try-except |
| `backend/models/gemini_adapter.py` | `import openai` → try-except |
| `backend/models/qwen_adapter.py` | `import openai` → try-except |
| `backend/models/wenxin_adapter.py` | `import openai` → try-except |

| `backend/voice/speech_to_text.py` | `import numpy as np` → try-except |
